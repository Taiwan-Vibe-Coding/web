Session 1–4 課前備註與實測對照

實測日：2026-07-12
身分：示範員工 WDL-E001–E050（實測多用座位 1＝王志明）
教室 UI：ChatGPT Work + 雲端 MCP wendell-dataverse-cloud
對象：講師（學員手冊仍以各 Session 練習原文為準）

Session 1｜認識 Codex（介面）

講師備忘

  • 確認每位學員都已安裝 ChatGPT Work（Codex）且帳號為 Plus（rate limit 問題）
  • 帶 HDMI 轉接頭；開場先說明左上角是 ChatGPT Work、教案仍稱 Codex 能力
  • 若有人安裝失敗，可先跟旁邊的人共用螢幕觀摩，下課後再處理
  • 練習時間是彈性的；時間緊時練習三可縮到 10 分鐘，但 延伸練習四必須進作業（文件分析缺口）
  • 重點是讓學員「親手體驗改寫任務描述後的差異」，這個 aha moment 比講完所有內容更重要
  • 課後作業含 RCFC＋文件／內容擇一，是 Session 2 開場分享材料

實測對照

預期成果（練習一～三）：詢價整理表、不合格通知信、日文來回信草稿；標出需人工確認處。

Session 2｜ERP／人資助理（MCP）

教室＝連 GCP，沒有本機伺服器。 列印檔（含 Bearer）：docs/_print/student-mcp-setup-card-PRINT.html（已 gitignore）。

講師備忘

  • 練習帳號：每人設定卡示範員工（WDL-E001–E050）——Bearer 綁定身分，get_my_payslips／請假／加班是該座位的資料
  • 權限邊界：list_employees 可查同事名單，但 get_my_payslips 不能指定別人的員編——只讀設定卡綁定的本人。學員若問「幫我查某某同事的薪資」，預期查不到別人的；順便講：自助工具＝本人，不是人資後台
  • 人資身分不是另開 Portal 登入。若 Not authenticated，先檢查設定卡是否用 Streamable HTTP 連對課堂 URL／Bearer（不要改設 STDIO／本機）
  • submit_leave 送出後是真的寫進 wendell-dataverse 這個 Dataverse demo 環境（不是正航／鼎新正式系統），所以看得到「已送出」的結果；跟學員說清楚的是「這是示範環境，不是公司正式紀錄」，不是「這是假動作、沒有真的執行」
  • 薪資單來源：課堂查到的 get_my_payslips 多半是 seed 示範資料。若後端已接 Payroll Engine（run_payroll／POST /v1/hr/payroll-runs），講師可另示範「試算 → 寫回 tw_payslips」——那才是合規算法路徑；不要把手動填的 demo 數字說成「系統依法規算出來的」
  • 如果 ERP／MCP 連線中斷：先確認 GCP（設定卡 URL、Bearer、場地外網）；再改口頭示範（照 session-02-exercises 預期回應）。教室不切本機 127.0.0.1:8020——那不是穩得上課路徑

實測對照

問法：請用 platform_health 查系統健康狀態…。同對話再查薪資。

可見：系統健康狀態正常，所有服務皆為 ok。
薪資：三張——PAY-2026-06（6 月）／PAY-2026-05／PAY-2026-04；6 月實發 NT$52,646.00（單號勿含 /，否則 get_my_payslip 會 404）

問法預期
我的請假紀錄？可能空清單或歷史；亦可查 absences
年假還剩幾天？邊界：無配額 API → 應說明查不到剩餘天數

Session 3｜公司資料整合（採購／O2C）

講師備忘

課前必做：

  • 跑一次 platform_health + list_customers；失敗才改口頭帶過
  • 確認 PO2026-001 狀態為 received 或 invoiced
  • 確認 AR／AP 查詢可用（練習三、四會用到）；失敗才準備替代說明
  • 延伸八 demo 能查到即可：list_projects（PRJ-EMI-2026-001）、list_sales_contracts（SC-2026-001）

示範帳號：今天查公司資料，不需特定員編（廠商／AP／客戶為共用示範資料）

若 ERP／MCP 連線中斷：改口頭示範，對照 exercises 預期結果帶過問法；教室不切本機

時間彈性：

  • 練習一若進展快，可提前進練習二
  • 練習時段可穿插短休息，但 15:40 必須收進討論與課後作業指派
  • 練習四若時間不夠，優先保留「有哪些客戶？」+ 一個委託案查詢
  • 選修延伸五～八（優先七：一頁分析）不能擠壓 15:40；時間到無論做到哪都收尾

實測對照

  • PO2026-001：狀態 received、預計到貨 2026-04-30、廠商全誠校正、金額 40,000 TWD。日期已過但已收貨 → 不是「逾期未交催貨」。
  • 雲端 MCP 廠商清單若夾到測試前綴資料，以示範錨點為準（如 PO2026-001／V003）。

Session 4｜AI 同事上線（複合查詢與寫入）

講師備忘

課前準備：

  • 印出練習三「凡事問AI」作答區與結業「部門 AI 工作手冊」空白頁，每人／每組一張（直接印 session-04-exercises 對應段即可）
  • 確認 AR 可用（練習一複合查詢可能用到）；失敗才準備替代說明
  • 準備一段「講師版 System Prompt」課堂對比用
  • 課前跑 platform_health；試建客戶＋報價各一次（皆 confirm=false、名稱加臨時前綴、不送出），確認寫入預覽可用
  • 開場再口頭強調姓名前綴：今天新建的客戶／廠商／單據名稱都要加自己的英文縮寫（例如 YL_測試客戶）

分組建議：

  • 少於 6 人：練習三、結業實作改個人完成後全班分享
  • 超過 10 人：練習三可兩人一組，發表時間壓到每組約 30 秒

時間彈性：

  • 練習一時段可微調，但不要排擠練習四寫入體驗
  • 15:40 結業手冊＋發表不可省略；選修 Track 不得往後擠
  • 選修 Track：A（全員）優先；B／C 依部門二選一；D（財務速覽）時間緊最先跳過。會議／Computer Use／CLI 見課程計畫文末補充教材，不佔 Track

實測對照

  • 練習四／延伸八：報價、訂單預覽需帶 product_name
  • 延伸練習七：採購單預覽需帶 name（與廠商 ID）
