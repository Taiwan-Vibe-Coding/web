認識 Codex：AI 幫你做事，不只是回答問題

認識 Codex／ChatGPT Work

日期：2026-07-21（一）13:00–16:00

時長：3 小時

對象：MIS + Key Users（混合技術 / 非技術背景）

工具：ChatGPT Work 桌面 app（舊稱 Codex；請先安裝並訂閱 Plus）

需要 ERP 連線：❌ 不需要（本堂為觀念 + ChatGPT Work／Codex 基本操作）

講師：陳育亮教授

課程目標

學員結束時能做到：

  1. 說出「agent（代理人）」和「聊天機器人」的根本差異
  2. 在 ChatGPT Work（Codex）建立 Project、開啟新任務、設定適當的模型等級，並知道 Personalization／Plugins／Scheduled 在哪裡
  3. 用 Codex 完成 3 個 EMC 業務情境任務，並能評估輸出品質與改寫任務描述（先寫滿 RCFC，不夠再加範例／分步）

各節詳細內容

13:00–13:15 開場（15 分鐘）

從學校到公司：把知識變成真的能用的東西

「幾位同學在世新的課上學了 AI 的概念和工具，覺得這對公司有幫助，就帶回來了。今天我們要做的事，就是把你們在課堂上學到的東西，真正用在穩得的業務上。不是繼續上課，是開始做事。」

切入問題（讓 MIS 學員開口，也讓其他同事有感）：「你們從世新帶回來之後，有沒有跟同事或主管分享過？他們的第一反應是什麼？」

這個問題的目的：拉近 MIS 和非 MIS 學員的距離，也讓大家說出「我原本對 AI 的期待和疑慮」。

Codex 的不同之處用一句話說：你不是在跟它聊天，你是在派任務給它。

13:15–13:45 概念：Chatbot vs Agent（30 分鐘）

兩者的根本差異

Chatbot（聊天機器人） Agent（代理人）
你給它 問題 任務
它做的事 回答 思考、拆解、執行、回報
需要你 每一步都問你 中間自己決定，完成後報告
範例 「CE 認證要做什麼測試？」 「幫我整理這份詢價單，生成一份 CE 報價草稿，存成表格」
能接工具嗎 ❌（純文字） ✅（Plugins 連 ERP、寄信、讀檔）

關鍵轉變：從「我來問，它來答」→ 「我來指派，它來執行」

什麼任務適合交給 Agent？

適合：

  • 有明確的輸入和輸出（「把這份規格整理成報價表」）
  • 步驟繁瑣但邏輯固定（「翻譯 + 整理 + 套格式」）
  • 需要查找、整合多個來源的資訊

不適合：

  • 需要人際判斷（「要不要答應這個客戶的殺價？」）
  • 涉及公司機密且沒有安全管控的情境
  • 結果需要100% 精確、無誤差（數字計算需人工驗核）

補充：RPA 不是同一件事嗎？

MIS 很可能被問到：「我們之前看過 RPA 業者，說可以自動化鼎新，跟這個有什麼不同？」

RPA（機器人流程自動化） AI Agent（如 Codex + MCP）
怎麼取資料 模擬滑鼠、畫面截取 直接呼叫 API / MCP 工具
ERP 換版或改介面 腳本全部重寫 Adapter 換掉，上層不動
能理解「意圖」嗎 ❌ 死板執行固定步驟 ✅ 可以理解模糊指令並拆解
需要 IT 維護 高（介面一變就壞） 低（API 穩定）
適合 重複性、畫面固定的操作 資料查詢、分析、報告、跨系統整合

RPA 是在 ERP 外面貼膠帶、讓機器人假裝成人在點畫面。
我們做的是讓 AI 直接讀懂資料、然後告訴你答案。

這不代表 RPA 沒用——截取發票圖片、自動填表這類任務 RPA 很合適。但查詢「這個月哪些客戶逾期」、「採購單審核進度」這類工作，AI Agent 搭配 MCP 更快、更穩、換 ERP 也不怕。

13:45–14:15 實機操作：認識 ChatGPT Work（Codex）介面（30 分鐘）

跟著做（所有學員同步操作）

Step 1：建立「穩得工作專案」

  1. 左側欄點「＋」新增 Project
  2. 點選「Start from scratch」
  3. 命名：穩得業務工具
  4. 說明一下 Project 的用途：把相關任務集中在同一個地方，AI 可以記住這個 Project 的背景

Step 2：認識「Do anything」輸入框

  • 這不是搜尋列，是「任務委託區」
  • 可以貼文字、上傳檔案（左下角 ＋）
  • 模型選擇（輸入框右下角）：
    • 課堂統一（首選）：GPT‑5.6 Terra，推理設 中等（Medium）
    • 做 Codex 教育訓練時，全班請統一用這個組合，方便對齊示範與排查問題。
    • 較難的分析／技術判斷必要時可改較高推理；本堂先不要各選各的

Step 3：第一個任務（全班一起）

在「穩得業務工具」Project 裡輸入：

你是穩得實業的業務助理，熟悉 EMC 測試服務。
任務：用條列方式說明客戶第一次詢價時，我們通常需要收集哪些資訊，
才能給出準確的報價？
格式：分類呈現（產品資訊、認證需求、時程需求），繁體中文。

觀察輸出，討論：它做到了嗎？哪裡可以改？

Step 4：Plugins、Scheduled、方案概念（跟做，約 8 分鐘）

桌面端左上角品牌為 ChatGPT Work（右方下拉即可看到 Codex）。左側有 New task／Scheduled／Plugins／Sites／Chat。

  • 方案：教室建議 Plus 以上（Free 額度容易在示範中途用完）。Free / Go / Plus / Pro / Business 差在模型存取與用量，開課前以 developers.openai.com/codex/pricing 為準。

- Plugins 市集（左側 Plugins）：標題「Work with ChatGPT across your favorite tools」。跟做確認 Documents／PDF 等已安裝；右上角齒輪 Manage 進入設定（Plugins／Apps／MCPs）。下堂課才接 wendell-dataverse-cloud MCP。

  • Scheduled（左側 Scheduled）：打開看「Daily brief／Weekly review／Follow-up monitor」範本即可，今天不建排程；Session 4 「凡事問AI」會寫「每週一自動摘要」這類需求。

14:15–14:25 Personalization（10 分鐘，跟做）

路徑：左側 Plugins → 右上角 Manage（齒輪）→ 左側 Personalization（或個人選單 → Settings → Personalization）。

跟做兩件事：

  1. Custom instructions 寫一行： 
    我是穩得實業同事，回覆請用繁體中文、條列優先。

    點選 Save，會看到螢幕上方出現 Saved agents.md 的提示。

  2. 打開 Enable memories（Memory (experimental) 區塊；若帳號有此開關）。
    另可見「Allow memory generation from tool-assisted tasks」（MCP／網搜任務也可寫入記憶）。

說明：Memory 會記住偏好與修正，跟 Session 4 的 Custom instructions／System Prompt 互補——Memory 偏自動累積，Custom instructions 偏你主動寫死角色。

今天先知道開關在哪；真正把語氣練穩，會在 Session 4 用 System Prompt 再做一次。

14:25–15:20 動手練習（55 分鐘）

詳見 session-01-exercises

  • 練習一（15 分鐘）：EMC 報價資訊整理
  • 練習二（20 分鐘）：測試不合格通知信（含語氣兩版）
  • 練習三（15 分鐘）：跨語言客戶溝通
  • 延伸練習四（優先選修／作業必做其一，20 分鐘）：文件比對與網路查證＋檔案預覽
  • 延伸練習五（選修，15 分鐘）：「一魚多吃」內容改寫（社群／電子報）——時間緊可當課後作業

練習規則：

  1. 先自己寫任務描述，送出看結果
  2. 評估輸出：哪裡好？哪裡不夠？
  3. 修改，再試一次
  4. 記錄改了什麼、差在哪

15:20–15:45 任務寫法的差異（25 分鐘）

為什麼寫法這麼重要

Codex 不會讀你心裡的想法。給它的任務描述越清楚，結果越接近你要的。

四堂課的第一個技巧：RCFC（技巧 1／4）

這四堂課會累積 4 個技巧，像疊積木一樣：今天的 RCFC 是任務描述的基本功，Session 2 會在它之上加「預覽 → 確認」的安全習慣，Session 3 加「一次問多件事」的整合提問，Session 4 用 System Prompt 把 RCFC 的「角色」「情境」直接固定住，不用每次重打。四堂課結束時，你會有一套完整、自己會用的工具箱。

四個關鍵要素（RCFC）

要素 說明 範例
角色 Role 告訴它扮演什麼角色 「你是穩得實業熟悉 FCC 認證的工程師」
情境 Context 給背景資料 「客戶是日系電子廠，產品是工業用無線模組」
格式 Format 指定輸出形式 「用表格，欄位：測試項目、標準、工時」
限制 Constraint 邊界條件 「只列 EMC 項目，不含安規，繁體中文」

現場對比

壞任務描述：

幫我寫報價單

好任務描述：

你是穩得實業的資深業務工程師。
客戶詢問：藍牙耳機，需要台灣 BSMI + 美國 FCC。
請幫我生成報價單草稿，表格格式，欄位包含：
認證別、測試項目、適用標準、預估工時、報價（填TBD）。
最後加注意事項：樣品數量與大約交期。繁體中文。

RCFC 與 Prompt 五技巧：同一套寫法，兩種講法

外面常講的「Prompt 五技巧」，跟今天的 RCFC 不是兩套互相競爭的方法——RCFC 是這四堂課要記住的任務描述骨架；五技巧只是把同一件事拆成更細的口訣。對照如下：

Prompt 五技巧（通用講法） 對應 RCFC 課堂怎麼記
1. 明確具體 Context＋Constraint 寫進 RCFC 就夠
2. 角色／受眾 Role 寫進 RCFC 就夠
3. 給範例（few-shot） （RCFC 沒點名） 額外補強
4. 要求分步 （RCFC 沒點名） 額外補強
5. 格式／語氣 Format（＋部分 Constraint） 寫進 RCFC 就夠

因此：先寫滿 RCFC（角色、情境、格式、限制）；若輸出還不穩，再加兩招——

補強 穩得情境怎麼寫
給範例 「照這個格式寫：〔貼一封你以前寫過的合格通知〕」
要求分步 「先列出風險點，等我確認後再寫完整信件」

任務委派思維（相對聊天問答）：Codex 是 Agent，常一次跑好幾步。把「做到什麼程度才算完成」講清楚，比只問一句話更重要。現場用同一任務做三次迭代：一句話 → 寫滿 RCFC → 再加範例（或分步）。

MIS 的新角色

以前 現在
業務說需求，MIS 開發 業務把任務交給 Codex，MIS 審核輸出
開發週期：數週 產出草稿：數分鐘
MIS 是瓶頸 MIS 是品質守門員

15:45–16:00 預告 Session 2 + Q&A（15 分鐘）

Session 2（7/24）重點預告

目前 Codex 只能用你「貼給它」的資料。下次我們要裝上 Plugin，讓它可以直接查詢系統資料——不過現階段接的是示範環境（wendell-dataverse，練習用的假資料），還不是鼎新或正航的正式資料；先讓大家熟悉「用 Codex 查系統」的操作方式，等 MIS 完成正航/鼎新的正式連線後才會換成真正的公司資料。

本堂講義重點（帶回去用）

RCFC 四要素速查卡

要素 問自己
Role 角色 我要 Codex 扮演誰？（職稱＋公司背景）
Context 情境 客戶／產品／狀況的關鍵背景是什麼？
Format 格式 我要表格、條列、還是一段話？欄位有哪些？
Constraint 限制 語言、範圍、不要做什麼？

不夠穩時再加：給範例、要求分步。

任務適不適合交給 Agent？

適合 不適合
輸入輸出明確（整理成報價表） 需要人際判斷（要不要答應殺價）
步驟固定但繁瑣（翻譯＋整理＋套格式） 涉及機密且無安全管控
需要整合多個來源資訊 結果必須 100% 精確、無誤差
