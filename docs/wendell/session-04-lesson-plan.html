讓 Codex 變成穩得自己的 AI 同事

AI 同事與凡事問AI

日期：2026-07-31（五）・結業

時長：3 小時

對象：全體學員（所有四堂都參加的）

工具：ChatGPT Work（Codex）+ 穩得 MCP 外掛（示範 ERP）

需要 ERP 連線：✅ 需要

講師：陳育亮教授

稱呼：ChatGPT Work＝App；Codex＝派任務能力；穩得 MCP 外掛＝連示範 ERP。

課程目標

學員結束時能做到：

  1. 讓 Codex 在一個對話裡跨多個工具做複合查詢
  2. 寫出一段讓 Codex 記住「我是穩得的誰、我需要什麼」的 System Prompt
  3. 為自己的部門設計一張「凡事問AI」，列出每週可以用 Codex 做的 3 件事
  4. 說出穩得 MCP／示範 ERP 的下一步，以及 MIS 的角色

15:30–15:40 穩得 AI 的下一步（10 分鐘）

現在有了什麼

  • 穩得 MCP 外掛（連示範 ERP）
    • 個人資料：請假、薪資、加班、費用報銷
    • 人事：員工查詢、出勤紀錄
    • 採購：廠商、採購單、採購發票、應付帳款、請購、收貨
    • 業務：客戶、委託案／訂單、報價單、應收帳款、銷售發票、折讓
    • 庫存：品項主檔、庫存水位
    • 財務：總帳科目、試算表、損益、資產負債表
    • 寫入：預覽 → 確認（客戶、報價、採購、請假／加班／報銷等）
    • 延伸：專案、銷售合約

接下來可以做什麼

以下依「先釐清、再擴量、後接真 ERP」排列，MIS 可當課後討論起點：

立刻可做（課後 2–4 週，不依賴換成正航／鼎新）：

  • 整理各部門「凡事問AI」清單（今天練習三／結業手冊的輸出）
  • 帳號與權限：誰能登入 ChatGPT Work、誰能對示範／正式環境做寫入
  • 年假配額／剩餘天數（Session 2 已點出缺口；可先在示範環境補齊，同仁每天最常用）

確認現況之後再做：

  • 接正航或鼎新（真實資料；上層 MCP 工具介面可沿用）
  • 餘額類查詢接到正式人事資料（年假剩餘、加班費累計）
  • 實驗室委託案現場進度（比訂單狀態更細的測項進度）

視需求（有真資料、有人願意維護再做）：

  • 自動化週報（Scheduled／每週一摘要）
  • 主管儀表板（公司整體一頁看）

MIS 的角色

在這個過程裡，MIS 不只是「幫大家裝軟體的人」。

MIS 是公司 AI 策略的守門人：

  • 哪些資料可以讓 Codex 存取？
  • 誰有權限查哪些資料？
  • 當 Codex 回傳錯誤資料時，流程是什麼？

這些決定，都需要 MIS 和業務單位一起討論。

給 MIS 的具體行動項目：

  1. 整理各部門的需求清單（今天練習三／結業手冊的輸出）
  2. 確認穩得目前是正航、鼎新，還是過渡中——有答案後再排真 ERP 接入
  3. 評估年假配額／剩餘天數要不要先補進示範或正式環境

A. 會議自動化 SOP（5 分鐘，可跟做）

準備一份會議逐字稿或錄音轉文字（講師提供範例即可）：

請依這份會議逐字稿產出：

2. 決議事項表（誰、做什麼、期限）

繁體中文。不要編造逐字稿沒有的承諾。

可再請 Codex：「把行動項目排成下週 Scheduled 提醒文案（只給文字，不要真的建立排程）」。

B. Computer Use（3 分鐘，講師展示或口述）

  • 概念：在 macOS 上讓 Codex 用游標操作畫面（開 App、點按鈕）。
  • 適合：重複畫面操作、示範流程錄製。
  • 風險：自主性高，出錯影響大——示範環境用、正式環境要 MIS 規範。

C. CLI／多終端機／平行任務（3 分鐘，口述）

  • CLI：終端機跑 codex，適合 MIS／會用命令列的同事批次任務。
  • 平行任務：同一時間開多個 Codex 任務（例如一邊整理報價、一邊查庫存）。
  • 什麼人需要：MIS、要大量改檔／跑腳本的人；一般行政／業務以桌面 App＋MCP 為主即可。
